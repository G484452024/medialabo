「Webコンテンツ作成」の成果

本テーマについて

学習目標は、

この授業では、HTMLでページの構造を記述し、CSSで装飾を行い、JavaScriptでページに動きを加えることで、Webページ作成の基本を理解することを目的としています。

プロジェクト課題について

プロジェクト課題では、与えられたAPIを用いて検索結果を取得し、それをWebページに表示するアプリケーションを作成しました。ユーザーの入力に応じて非同期通信を行い、取得したデータをDOMに動的に追加・更新する実装に挑戦しました。特に、ボタンのイベント処理や、エラー処理、レスポンスの整形に苦労しました。

プロジェクト課題のページ：project.html

数当てゲーム

乱数で 1〜10の値を1つ決めて，「答え」とする.ユーザは答えを予想して，数を最大で3回入力できるユーザの入力が外れた場合は，入力よりも答えが大きいか小さいか表示する.3回以内に答えの数を当てればユーザの勝ち，そうでなければ負けページには，テキストボックスとボタンが設置されている.ユーザはテキストボックスに予想した数を入力ボタンを押すと判定現時点ではテキストボックスからの入力やボタンによるイベントの処理，結果をページに表示する方法などのプログラム

数当てゲームのページ:kazuate.html

各授業回の学習事項

第1回WWWとHTML

HTML は Web ページの構造を記述するための書式です。HTML 文書を作ることによって、Webページに掲載するコンテンツと、ページの基本的なレイアウトを設定できます。ページの作成の仕方からURLの貼り方、HTML要素、ブロック要素とインライン要素、空要素、属性、HTMLの特殊文字、HTML文書のコメントなどを作れるか試した。

演習の成果:演習1-4HTMLのさまざまな要素

第2回CSSによるWebページの装飾

HTML は要素の集まりでできています。この要素それぞれの見た目の設定（装飾）について今回学んだ。ページを装飾する記述は CSS と言います。CSS は要素の見た目を変更するものなので、HTML が最初に与えられて、次にその装飾をする。CSSを記述する。CSSを追加し、絵文字の追加やフォントを変更、カラーの変更、リンクの色設定などを作り個性を出せるように作った。

演習の成果:演習2-3ウェブブラウザ説明ページの装飾

第3回JavaScriptの基本

JavaScript は Web ページ上で複雑な機能を実現するプログラミング言語.Web ページを開いた直後や，ボタンを押したときに JavaScript プログラムが実行されます．

演習の成果:演習3-6

第4回DOMとJavaScript

ボタンを押したらページが変化する」処理を JavaScript プログラムとして記述する方法,3のDOM（Document Object Model）は Web コンテンツを表現するデータ構造です．DOM を使用することによって，JavaScript などのプログラムによって，Web コンテンツの内容を読み取ったり，コンテンツに変更を加えたりする処理が書けるようになります．

演習の成果:演習4-10

第５回イベント処理

DOM（Document Object Model）は Web コンテンツを表現するデータ構造です．DOM を使って次のことも行えます．要素の新規作成,要素の編集（要素の属性を追加したり変更したり）,要素の削除,今回はこれらの DOM の機能を使ってみます．今回の内容をマスターすると，JavaScript を通じてページを自由に変更できるようになります．

演習の成果演習5-7

工夫点，苦心点，強調点，感想

  • 今回の授業で特に工夫した点は、HTML・CSS・JavaScriptをバランスよく活用し、見やすく使いやすいWebページを目指したことです。具体的には、HTMLでは意味のある要素を使って文書構造を明確にし、CSSではカラーや余白を調整して視認性を高めました。また、ボタンやテキストボックスの配置にも注意を払い、ユーザーが直感的に操作できるようにしました。JavaScriptでは、イベントリスナーを使って動的な動作を実現し、ユーザーが入力した値に応じてメッセージが変化するように工夫しました。数当てゲームでは、残り回数やヒント表示などのインターフェースも考慮し、ユーザーが飽きずに楽しめる仕組みにしました。
  • 最も苦労したのは、JavaScriptによるDOM操作とイベント処理の実装です。特に、「数当てゲーム」では、ユーザーの入力を正しく取得し、それに応じた条件分岐を実現するところで何度もエラーに悩まされました。入力値が数値でない場合や、ボタンを複数回押したときの挙動など、予期せぬ動作が起きるケースに対応するために、parseInt()やisNaN()を使って入力チェックを行うようにしました。また、JavaScriptの変数スコープや、イベントリスナーのタイミングにも注意が必要で、デバッグ作業を繰り返しながら少しずつ動作を安定させていきました。CSSについても、思い通りのレイアウトにならず、Flexboxのプロパティを調整したり、メディアクエリを試したりと、多くの試行錯誤を重ねました。
  • このレポートで特に強調したい点は、基本的な文法だけでなく「実際に使えるページを自分の力で組み立てることができた」という点です。授業で学んだ技術を単に模倣するだけでなく、自分なりに応用しながらページを作成できたことが大きな成果だと感じています。また、Web制作では「見た目」と「機能」の両方が重要であることを実感しました。CSSでページのデザインを整え、JavaScriptでインタラクションを追加することで、静的なHTMLが一気に動的で魅力的なコンテンツに変化します。コードの書き方だけでなく、ユーザー目線で使いやすさやわかりやすさを意識できたことは今後の学びにも大いに役立つと考えています。
  • この授業を通じて、Webページ作成の基礎から応用までを段階的に学ぶことができ、とても有意義な経験になりました。最初はHTMLやCSSの構文すら曖昧で、画面に表示される仕組みもよく理解できていませんでしたが、演習やプロジェクト課題を通じて、ページ全体を設計・構築する力が身についたと感じます。特にJavaScriptについては、苦手意識がありましたが、課題を通じて少しずつ扱えるようになり、プログラミングの楽しさも知ることができました。今後はこの授業で学んだことを活かして、より高度なページやWebアプリケーションの作成にも挑戦していきたいです。また、自分で考えて解決する力の大切さを改めて実感したので、これからも試行錯誤を恐れずに取り組んでいきたいと思います。
